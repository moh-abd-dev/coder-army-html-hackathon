Overview

Forms are a vital part of the web, allowing users to input and submit data. The main container for all form elements is the <form> tag. Inside this, various <input> tags are used to collect different types of information.

A crucial element for accessibility and usability is the <label> tag. It provides a description for an input field. By linking a label to an input using the for attribute (which should match the input's id), users can click on the label's text to immediately focus on the input field, which is especially helpful on mobile devices.

Imp Form Elements

Text-Based Inputs

These are used for collecting written information.

  • <input type="text"> : Creates a single-line text field for general input like names or usernames.
  • <input type="password"> : A text field that hides the characters as the user types, used for sensitive information.
  • <input type="email"> : A field designed for email addresses. Browsers can automatically check if the entered text is in a valid email format.
  • <textarea> : A multi-line text field, ideal for longer messages, comments, or addresses.

Choice Elements

These allow users to select from a predefined list of options.

  • <input type="radio"> : Radio buttons are used when a user must select only one option from a group (e.g., Yes/No). All radio buttons in a single choice group must share the same 'name' attribute.
  • <input type="checkbox"> : Checkboxes are used when a user can select zero, one, or multiple options from a list (e.g., selecting hobbies or topics of interest).

Buttons

Buttons trigger actions within the form.

  • <button type="submit"> : When clicked, this button attempts to send the collected form data to a server.
  • <button type="reset"> : This button clears all the form fields and returns them to their default state.
