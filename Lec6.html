Media Tags

You can embed audio and video directly into your webpages to make them more engaging.

The <audio> Tag

This tag is used to embed sound files. The src attribute points to the location of the audio file (like an .mp3). You must include the controls attribute, which tells the browser to display the default audio player controls like play, pause, and volume.

  • Examples:

The <video> Tag

This tag is for embedding video files. Similar to the audio tag, it uses an src attribute for the video file path and requires the controls attribute to show the player. You can also set the width and height of the video player to reserve space for it on the page.

  • Examples:

The <iframe> Tag

The <iframe> tag is a powerful tool that lets you embed an entire external webpage into your own. Its most common use is for embedding YouTube videos. You don't need to write this code yourself; simply go to the YouTube video, click 'Share', then 'Embed', and copy the code provided.

    • Multi-Page website

    A multi-page website is a collection of individual HTML files linked together.

    Key Principles

    • Consistent Structure: Every page should share a common structure, typically including the same <header> (with navigation) and <footer>. This creates a predictable and user-friendly experience.
    • Root File (index.html): By web server convention, the main homepage of any website should be named index.html. When you navigate to a domain (like www.google.com), the server automatically looks for and serves the index.html file. For example, a file you might have named home.html should be renamed to index.html to act as the site's main page.
    • Organized File Paths: Keep your project tidy. Store all your images in an images folder, your CSS files in a styles folder, and so on. This makes managing your site much easier as it grows.

    Website Deployment

    When you build a website on your computer, it runs locally. This means only you can access it, and the file path in your browser isn't a shareable link. To make your site accessible to everyone on the internet, you need to deploy it. Deployment is the process of publishing your local website files to a web server.

    So, let's see how we can deploy using Netlify.

    • Netlify is a very popular platform for web developers. You can simply drag and drop your entire website folder onto their dashboard, and it will be live in seconds. It also connects directly to GitHub.
    • After deploying, the platform will give you a public URL (e.g., your-project-name.netlify.app) where your live website can be viewed by anyone in the world.
